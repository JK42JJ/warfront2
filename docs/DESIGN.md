## 0. 설계 원칙 (우선순위 순)

0. **★생각 먼저, 구현은 그 다음 (Think-First)** — 툴의 존재 이유(James 2026-07-20): "스스로 생각하고 솔루션을 머릿속으로 떠올리며(또는 연필로 구조를 그려보고) → 그것을 코드로 구현. 이 흐름이 자연스러워지도록 반복 훈련, 그 과정에서 재미."
   구현: 모든 문제는 **THINK 게이트**를 통과해야 에디터가 열린다 — ① 유형·자료구조·복잡도를 먼저 선언(한 줄 타이핑) 또는 ✏️ 연필 모드(타이머 일시정지, 종이에 구조 그리기) → ② 에디터 해금 → ③ 채점 후 **내 설계 선언 vs 모범 접근 비교** 피드백. "설계 적중" 보너스로 게임화 — 생각이 맞아떨어지는 쾌감이 재미의 원천.
1. **타건감** — 키 입력마다 즉시 반응(글자 단위 색 피드백). 마우스 제로. 손이 홈로우를 떠나지 않는다.
2. **코딩 자체에만 집중** — 한 화면 한 목적. 설치 1줄, 실행 1줄(`wf`). tmux·브라우저 등 외부 의존 제거.
3. **중독성 = 즉시 피드백 루프** — 저장→채점→결과가 1초 안에. 스트릭·개인최고·랭크가 매 세션 갱신.
4. **큰 텍스트·직관** — 대형 숫자(Digits), 여백, 상태는 색·기호로. 읽는 화면이 아니라 보이는 화면.
5. **과학적 반복** — 감이 아니라 FSRS(간격반복)가 "오늘 뭘 복습할지" 결정. 잊을 때쯤 다시 나온다.

## 2. 기술 스택 (리서치 #1 실검증 근거, 2026-07-20)

| 계층 | 선택 | 근거(1차 출처 검증) |
|---|---|---|
| UI | **Textual 8.x** | 36.7k★, 2026-06 릴리스 활발. **smassh**(MonkeyType 클론 2k★)가 Textual로 구현 = 타이핑 게임 반응성 실증. Digits(대형숫자)·TextArea.code_editor(구문강조·줄번호·undo)·TCSS·애니메이션 내장 |
| 대형 배너 | textual-pyfiglet | ASCII 배너+그라데이션 애니메이션 |
| 반복 스케줄 | **fsrs 6.x** (py-fsrs) | Anki 공식 채택 알고리즘. 30일 학습+20일 체득이 파라미터로 구현(desired_retention=0.9, max_interval≈20d 캡) |
| 채점 | **pytest + subprocess(timeout) → results.json** | exercism python-test-runner 구조 차용. 로컬 자기학습 = 무거운 샌드박스 불요 |
| 저장 | SQLite (단일 파일 `~/.warfront2/wf.db`) | 카드·리뷰로그·세션·스트릭·개인최고 |
| CLI 진입 | typer | `wf` 한 명령 |
| 외부에디터 모드 | watchdog + App.suspend() | `e`키로 vim 탈출(API 검증), 복귀·저장 시 자동 채점 |

Python ≥3.10. 의존성 6개 — 가볍게 유지.

## 4. ⌨️ 타건감 설계 (제1 원칙의 구현)

### 4-1. 훈련 모드 3단 (무술 카타 원리 — 타이핑이 곧 체화)
| 모드 | 무엇 | 타건 메커니즘 |
|---|---|---|
| **① 따라치기 (guided)** | 패턴 코드(BFS 뼈대 등)를 화면에 띄우고 그대로 타이핑 | 글자 단위 즉시 색 판정(회색→흰색=정타, 빨강=오타). 실시간 WPM·정확도 대형 표시 |
| **② 빈칸 채우기 (cloze)** | 같은 패턴에서 핵심 줄만 빈칸 — 빈칸에서만 손이 멈춘다 | 빈칸 진입 시 하이라이트, 채우면 즉시 판정음(시각 플래시) |
| **③ 백지 재현 (recall)** | 문제 설명만 주고 패턴 전체를 기억으로 타이핑 | 완성 후 diff 표시 — 근육기억 최종 검증 |

**진급**: ①→②→③은 정확도 95%+ 시 해금. 같은 패턴이 FSRS 주기로 재등장할 땐 더 높은 모드로.

### 4-2. 중독 루프 (실증 UX 패턴 10, 리서치 #1)
- **세션 시작 마찰 제로**: `wf` → 엔터 한 번 → 오늘의 첫 카드. (오늘의 챌린지 패턴)
- **키 입력 즉시 피드백**: ttyper식 글자 색 판정 — 늦은 채점 없음.
- **고스트 대결**: 같은 카타의 과거 내 기록이 유령 커서로 함께 달림(smassh 검증 패턴).
- **실패 임계치(선택 모드)**: 정확도 90% 미달 시 카타 즉시 리셋 — 긴장감.
- **대형 숫자 결과**: 세션 끝 = Digits로 WPM·정확도·연속일 크게. 개인최고 갱신 시 배너.
- **스트릭 + 주간 히트맵**: 홈 화면 상단 고정.
- **벨트 승급**: 유형별 흰띠→…→검은띠(카타 ③모드 통과 + 실전문제 N개).
- **난이도 타이머**: 실전 문제에 프로그래머스식 제한시간 — 카운트다운 표시.

## 5. 채점 엔진

- 문제 = `README.md`(지문) + `test_XX.py`(pytest 케이스, 숨김 케이스 포함) + `answer.py`(모범답안·해설).
- 실행: `subprocess.run([python, -m, pytest, ...], timeout=10s)` → JSON 리포트 → 케이스별 ✅/❌ + 실패 케이스의 입력/기대/실제 표시.
- 부분 점수: 케이스 통과율 표시(실전 코테 모사).
- 시간복잡도 게이트: 대형 입력 케이스에 타임아웃 — "맞았는데 느림"을 실전처럼 fail 처리.

### 6-4. 실전 모사 요건 (리서치 #2 실환경 조사)
1. **입출력 2모드**: solution 함수 완성형(프로그래머스식) + 표준입출력/T케이스(백준·구름·삼성식) — 문제마다 실전 그대로.
2. **히든 테스트케이스**: 예제만 통과 ≠ 정답 경험 재현.
3. **정확성/효율성 분리 채점**: 대형 입력 케이스 타임아웃 = "맞았는데 느려서 탈락" 실전 경험.
4. 채점 옵션: 케이스 비율 점수(표준) + 세트 all-or-nothing(카카오식).
5. 타이머 모의: 120분/3(표준) · 180분/2(삼성형).

### 6-5. 문제 소스 전략 (저작권 준수 — 리서치 확인)
- **코어는 자작 문제**(지문+테스트+해설 전부 자체 제작, SRE/실생활 소재) — 백준·프로그래머스 지문은 **복제 금지**(이용규칙·ToS 확인됨).

# warfront2

터미널에서 쓰는 코딩테스트 훈련 도구입니다. 50일 과정이고, 문제를 풀기 전에
접근을 먼저 적게 만드는 것이 특징입니다.

## 왜 이렇게 만들었나

코테를 준비하면서 두 가지 문제가 있었습니다. 풀이를 눈으로만 읽고 넘어가면
며칠 뒤에 기억이 안 난다는 것. 그리고 문제를 보자마자 코드부터 치기 시작하면
중간에 길을 잃는다는 것.

그래서 이렇게 동작합니다.

접근을 한 줄이라도 먼저 적어야 에디터가 열립니다. 머릿속으로든 연필로든
풀이 구조를 그린 다음에 코드를 치라는 뜻입니다.

같은 패턴을 네 단계로 반복합니다. 보고 치고, 빈칸을 채우고, 안 보고 다시 쓰고,
마지막에는 처음부터 직접 구현합니다. 각 단계는 정확도 95%를 넘겨야 열립니다.

직접 구현 단계는 글자 비교가 아니라 테스트로 채점합니다. 변수명이 달라도
동작이 맞으면 통과입니다. 느린 코드는 큰 입력에서 시간 초과로 떨어집니다.
실제 채점 환경과 같은 방식입니다.

막히면 F1으로 힌트를 봅니다. 처음엔 구조(어떤 순서로 만드는지)만 보여주고,
한 번 더 누르면 현재 줄이 왜 그렇게 생겼는지 설명합니다. F2는 알고리즘이
동작하는 모습을 그림으로 보여줍니다. BFS라면 탐색이 퍼져나가는 과정이
단계별로 그려집니다.

문제는 패턴 15개로 시작합니다. 구현, 해시, 그리디, 스택, 힙, 백트래킹, BFS,
DFS, 문자열, DP, 이분탐색, 투포인터, 다익스트라. 한국 기업 코테 출제 빈도가
높은 순서대로 배치했습니다. 목표선은 프로그래머스 Lv2를 안정적으로 푸는
수준입니다.

## 설치

Python 3.10 이상과 git이 필요합니다.

```bash
git clone https://github.com/JK42JJ/warfront2.git
cd warfront2
pip install -e .
wf
```

포크해서 쓰려면 본인 포크를 clone하면 됩니다.

macOS는 기본 터미널, iTerm2, Ghostty 어디서든 됩니다. Windows는
Windows Terminal에서 `py -m pip install -e .`로 설치하세요. `wf` 명령이
안 잡히면 `py -m wf.cli`로 실행할 수 있습니다.

## 명령

```
wf                  훈련 시작
wf update           최신 문제와 기능 받기
wf version          버전과 문제 수 확인
wf setup <repo>     훈련 기록을 내 GitHub repo로 백업·복원
wf sprint           속성 7일 모드 (해제: --off)
wf reset            기록 초기화
```

## 유형 인식 드릴

문제를 외워도 시험장에서 못 푸는 이유는 간단합니다. 시험 문제는 알고리즘이
아니라 이야기로 나오는데, 문제집은 챕터 제목이 유형을 미리 알려주니 판별
연습을 한 번도 못 한 채로 풀이만 연습하게 되기 때문입니다.

대시보드에서 `i`를 누르면 지문만 보고 유형을 고르는 드릴이 시작됩니다.
문항 50개 전부가 실제 기출(프로그래머스 Kit, 카카오 공식 해설, 삼성 기출,
HackerRank Interview Kit)에서 온 것이고, 보기에는 헷갈리기 쉬운 유형이
함정으로 섞여 있습니다. 가중치가 있으면 BFS가 아니라 다익스트라라는 식의
경계를 몸으로 익히는 게 목적입니다. 인식률 90%가 목표선입니다.

## 갑자기 코테가 잡혔을 때

50일이 없을 때는 속성 모드를 씁니다. 대시보드에서 `t`를 누르거나
`wf sprint`를 실행하면 7일 압축 과정이 시작됩니다. 출제 빈도가 가장
높은 유형만 골라 하루 60~90분씩, 마지막 날은 전체 리허설입니다.
그날 할 카타에 ⚡ 표시가 붙습니다.

## 훈련 기록을 GitHub에 남기기

GitHub에 빈 private repo를 하나 만들고 연결하면, 세션이 끝날 때마다
그날의 기록(세션 수, 시간, 푼 문제, 정확도)이 자동으로 커밋됩니다.

```bash
wf setup https://github.com/<계정>/my-records.git
```

DB 원본은 로컬에만 남고, 날짜별 요약 JSON만 올라갑니다. push가 실패해도
훈련은 그대로 진행되고 다음 세션 때 다시 시도합니다.

## 훈련 중 키

```
Enter     다음 단계 자동 선택
g b r s   단계 직접 선택 (보고 / 빈칸 / 재현 / 구현)
F1        힌트
F2        동작 그림
Ctrl+R    채점 (구현 단계)
ESC       대시보드로
```

설계 배경과 근거 자료는 [docs/DESIGN.md](docs/DESIGN.md)에 있습니다.
